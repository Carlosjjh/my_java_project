# 前端、后端、数据库交互说明

本文档说明当前商城项目中前端、后端、数据库三者之间如何交互，以及项目中是否使用了 API。

## 一、三者分别是什么

### 前端

前端是用户在浏览器里看到和操作的页面。

本项目中的前端文件：

```text
src/main/resources/static/index.html
src/main/resources/static/styles.css
src/main/resources/static/app.js
```

作用：

- `index.html`：页面结构，例如商品区域、购物车区域、订单表单。
- `styles.css`：页面样式，例如布局、颜色、按钮、商品卡片。
- `app.js`：页面交互逻辑，例如加载商品、加入购物车、提交订单。

### 后端

后端是运行在服务器上的 Java 程序，负责接收前端请求、处理业务逻辑、读写数据库。

本项目中的后端核心文件：

```text
src/main/java/carlos/jiang/App.java
src/main/java/carlos/jiang/web/ProductController.java
src/main/java/carlos/jiang/web/OrderController.java
src/main/java/carlos/jiang/service/OrderService.java
src/main/java/carlos/jiang/repository/ProductRepository.java
src/main/java/carlos/jiang/repository/ShopOrderRepository.java
src/main/java/carlos/jiang/model/Product.java
src/main/java/carlos/jiang/model/ShopOrder.java
src/main/java/carlos/jiang/model/OrderItem.java
```

作用：

- `Controller`：提供接口，接收前端 HTTP 请求。
- `Service`：处理业务逻辑，例如创建订单、扣减库存。
- `Repository`：负责访问数据库。
- `Model`：定义数据库中的数据结构。

### 数据库

数据库负责保存商品、订单、订单明细等数据。

本项目使用的是 H2 内存数据库：

```text
jdbc:h2:mem:mall
```

配置文件：

```text
src/main/resources/application.properties
```

主要数据表：

```text
PRODUCT
SHOP_ORDER
ORDER_ITEM
```

注意：当前使用的是内存数据库，项目停止后数据会清空，重新启动时会再次初始化示例商品。

## 二、有没有使用 API

有，本项目使用了 API。

这里的 API 指的是后端提供给前端调用的 HTTP 接口。前端不会直接访问数据库，而是通过 API 向后端要数据或提交数据。

当前项目中的 API 有：

```http
GET /api/products
GET /api/products/{id}
POST /api/orders
GET /api/orders
```

这些 API 在后端 Controller 中定义：

```text
src/main/java/carlos/jiang/web/ProductController.java
src/main/java/carlos/jiang/web/OrderController.java
```

前端通过 JavaScript 的 `fetch()` 调用这些 API：

```text
src/main/resources/static/app.js
```

## 三、前端和后端如何交互

前端和后端通过 HTTP API 交互。

浏览器打开页面后，会加载：

```text
http://localhost:8081
```

Spring Boot 会自动返回：

```text
src/main/resources/static/index.html
```

页面加载完成后，`app.js` 会调用后端接口。

### 加载商品

前端执行：

```javascript
fetch("/api/products")
```

请求后端：

```http
GET /api/products
```

后端由 `ProductController` 接收请求：

```java
@GetMapping
public List<Product> findAll() {
    return productRepository.findAll();
}
```

后端返回商品 JSON 数据，前端拿到后渲染商品卡片。

交互流程：

```text
浏览器页面
  -> app.js 调用 fetch("/api/products")
  -> ProductController
  -> ProductRepository
  -> H2 数据库中的 PRODUCT 表
  -> 返回商品 JSON
  -> app.js 渲染商品列表
```

### 提交订单

用户在前端操作：

```text
加入购物车 -> 填写姓名、手机号、地址 -> 点击提交订单
```

前端会整理出订单数据，例如：

```json
{
  "customerName": "张三",
  "phone": "13800000000",
  "address": "上海市浦东新区",
  "items": [
    {
      "productId": 1,
      "quantity": 2
    }
  ]
}
```

然后调用：

```javascript
fetch("/api/orders", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(payload)
})
```

请求后端：

```http
POST /api/orders
```

后端由 `OrderController` 接收请求，再交给 `OrderService` 处理：

```text
OrderController -> OrderService
```

`OrderService` 会做这些事情：

- 检查商品是否存在。
- 合并购物车中相同商品。
- 检查库存是否足够。
- 扣减商品库存。
- 创建订单。
- 创建订单明细。
- 保存到数据库。

交互流程：

```text
浏览器页面
  -> app.js 提交订单 JSON
  -> OrderController
  -> OrderService
  -> ProductRepository 查询商品和库存
  -> ShopOrderRepository 保存订单
  -> H2 数据库写入 SHOP_ORDER 和 ORDER_ITEM
  -> 返回订单 JSON
  -> app.js 显示下单成功
```

## 四、后端和数据库如何交互

后端不直接手写 SQL，而是使用 Spring Data JPA 操作数据库。

本项目中的 Repository：

```text
ProductRepository
ShopOrderRepository
```

它们继承了：

```java
JpaRepository
```

因此可以直接使用常见数据库操作方法：

```java
findAll()
findById()
save()
count()
```

例如商品查询：

```java
productRepository.findAll();
```

实际含义是：

```text
查询 PRODUCT 表中的所有商品
```

例如保存订单：

```java
orderRepository.save(order);
```

实际含义是：

```text
向 SHOP_ORDER 表和 ORDER_ITEM 表写入订单数据
```

后端和数据库的关系：

```text
Controller
  -> Service
  -> Repository
  -> JPA/Hibernate
  -> H2 数据库
```

其中 JPA/Hibernate 会根据实体类自动生成和执行 SQL。

## 五、前端和数据库是否直接交互

不会。

前端不能直接访问数据库，也不应该直接访问数据库。

原因：

- 数据库账号密码不能暴露给浏览器。
- 前端直接操作数据库不安全。
- 业务规则应该由后端统一控制，例如库存校验、订单金额计算。
- 后端可以统一处理异常、权限、数据校验。

所以正确关系是：

```text
前端 -> 后端 API -> 数据库
```

而不是：

```text
前端 -> 数据库
```

## 六、数据库数据如何进入前端页面

以商品列表为例：

1. 应用启动时，`DataInitializer` 自动创建示例商品。
2. 商品数据保存到 H2 的 `PRODUCT` 表。
3. 浏览器打开首页。
4. `app.js` 调用 `GET /api/products`。
5. 后端从 `PRODUCT` 表查询商品。
6. 后端把商品转换成 JSON 返回给前端。
7. 前端把 JSON 渲染成商品卡片。

流程图：

```text
DataInitializer
  -> PRODUCT 表
  -> ProductRepository.findAll()
  -> ProductController
  -> GET /api/products
  -> app.js
  -> 商品卡片
```

## 七、订单如何保存到数据库

用户提交订单后：

1. 前端把购物车和收货信息提交给 `POST /api/orders`。
2. 后端 `OrderController` 接收请求。
3. `OrderService` 查询商品并扣减库存。
4. 创建 `ShopOrder` 订单对象。
5. 创建 `OrderItem` 订单明细对象。
6. 调用 `ShopOrderRepository.save(order)`。
7. JPA 把数据写入数据库。

写入的数据表：

```text
SHOP_ORDER
ORDER_ITEM
PRODUCT
```

其中：

- `SHOP_ORDER` 保存订单主信息。
- `ORDER_ITEM` 保存订单中的商品明细。
- `PRODUCT` 更新商品库存。

## 八、如何查看 API 返回的数据

启动项目：

```powershell
mvn spring-boot:run
```

访问商品 API：

```text
http://localhost:8081/api/products
```

访问订单 API：

```text
http://localhost:8081/api/orders
```

也可以在浏览器开发者工具中查看：

```text
F12 -> Network -> Fetch/XHR
```

在这里可以看到前端调用了哪些 API、请求参数是什么、后端返回了什么。

## 九、如何查看数据库中的订单

打开 H2 控制台：

```text
http://localhost:8081/h2-console
```

连接信息：

```text
JDBC URL: jdbc:h2:mem:mall
User Name: sa
Password: 留空
```

查询订单主表：

```sql
SELECT * FROM SHOP_ORDER;
```

查询订单明细：

```sql
SELECT * FROM ORDER_ITEM;
```

联合查看订单和明细：

```sql
SELECT
  o.ID AS ORDER_ID,
  o.CUSTOMER_NAME,
  o.PHONE,
  o.ADDRESS,
  o.TOTAL_AMOUNT,
  o.CREATED_AT,
  i.PRODUCT_NAME,
  i.UNIT_PRICE,
  i.QUANTITY,
  i.LINE_TOTAL
FROM SHOP_ORDER o
JOIN ORDER_ITEM i ON i.ORDER_ID = o.ID
ORDER BY o.ID DESC;
```

## 十、总结

本项目的核心交互关系是：

```text
前端页面
  -> 调用后端 API
  -> 后端处理业务逻辑
  -> 后端通过 JPA 读写 H2 数据库
  -> 后端返回 JSON
  -> 前端更新页面
```

所以，本项目确实使用了 API。API 是前端和后端之间的桥梁，数据库则由后端通过 JPA 间接访问。
